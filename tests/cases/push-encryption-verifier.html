Implementing support for Web Push Encryption can be tricky as it involves a lot of steps. This tool will tell you the intermediate values based on the input keys, which should aid step-by-step debugging in case it's not working.

Input variables

The keys and auth secret must either be base64url encoded or hex values. Public keys must be given as P-256 uncompressed EC points, and the private key must be given as ASN.1 PrivateKeyInfo.

NIST P-256 uncompressed EC points

A series of 65 bytes starting with 0x04 followed by two 32-byte points, per SEC1 2.3.3. The format given by the W3C Push API's PushSubscription.getKey() method.

When using Web Crypto, it can be derived by exporting the public key as JWK, then concatenating [0x04, base64url_decode(jwk.x), base64url_decode(jwk.y)].

ASN.1 PrivateKeyInfo

A series of at least ?? bytes per the PrivateKeyInfo definition in RFC 5208. The format given by the Web Crypto API when exporting a private P-256 key in "pkcs8" format.
